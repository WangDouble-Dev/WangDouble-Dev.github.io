IDE|PHPStorm Power Save Mode模式开启导致代码不提示及自动补全失效

Wang Double

February 25, 2024

其它

某天用PHPStorm写代码的时候，代码突然不提示了,不会自动补全及导入类了。代码编写速度直线下降。

检查了很多设置，最后发现是IDE的Power Save Mode模式开启了。

通常，IDE（集成开发环境）的省电模式是为了在使用电池供电时减少系统资源的使用，以延长电池寿命。省电模式通常涉及一系列设置和调整，以降低软件的性能需求，从而减少能耗。这些调整可能包括：

  1. 自动保存设置： 将自动保存的频率调整为更长的时间间隔，以减少IO操作。

  2. 自动代码分析： 降低代码分析的频率或关闭一些耗费资源的代码检查功能。

  3. 自动代码补全： 可能会降低或禁用自动代码补全功能，因为它通常需要消耗一定的计算资源。

  4. 实时语法检查： 减少或关闭实时语法检查，以降低对系统资源的占用。

  5. 背景任务： 延迟或限制一些后台任务的执行，以减少对CPU和内存的需求。

  6. 界面动画和效果： 禁用一些界面动画和特效，以减少对图形处理单元（GPU）的负担。

  7. 插件管理： 停用不必要的插件，因为一些插件可能在后台执行任务，增加了系统负担。

  8. 缓存策略： 调整缓存策略，减少缓存刷新的频率。

关闭省电模式方法：

  • 通过点击IDE的右下角的电源图标，关闭Power Save Mode模式。
  • 也可以通过File -> Power Save Mode菜单项来切换Power Save Mode模式。
